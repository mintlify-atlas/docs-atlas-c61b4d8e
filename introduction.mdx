Project Based Learning is a community-curated collection of programming tutorials where you learn by building real applications from scratch. Rather than passively reading documentation or watching lectures, you construct something tangible — a web app, a compiler, an operating system, a game — and absorb the language or technology as a natural side effect of doing. With over 271,000 GitHub stars and contributions from developers around the world, this list has grown into one of the most trusted starting points for hands-on learning, spanning 20+ programming languages and dozens of technology stacks.

Every tutorial in this collection results in a working project. Browse by the domain that interests you most, then dive into the language you want to learn.

## Who Is This For?

Project Based Learning serves developers at every stage. Whether you are picking up your first language or your tenth, there is a project here sized to your current skill level.

<Tabs>
  <Tab title="Beginner">
    If you are new to programming or exploring a language for the first time, start with something you will actually enjoy finishing. Good entry points include:

    - **JavaScript**: [Build 30 things in 30 days with 30 tutorials](https://javascript30.com) — small, self-contained projects perfect for building confidence.
    - **Python**: [Create a Blog Web App in Django](https://tutorial.djangogirls.org/en/) — a gentle walkthrough that results in a deployed web application.
    - **HTML/CSS**: [Build A Loading Screen](https://medium.freecodecamp.org/how-to-build-a-delightful-loading-screen-in-5-minutes-847991da509f) — visual feedback in minutes.

    The key for beginners is choosing a project whose output excites you. If you want to see a game on screen, start with a game tutorial. If you want a website, start there.
  </Tab>
  <Tab title="Intermediate">
    If you already know one language and want to branch out — or deepen your understanding of a language you use day to day — look for tutorials that involve multiple technologies or tackle a non-trivial problem:

    - **Go**: [Building Blockchain in Go](https://jeiwan.net/posts/building-blockchain-in-go-part-1/) (7-part series) — teaches Go idioms through a real distributed-systems concept.
    - **Python**: [Build a Simple Blockchain in Python](https://hackernoon.com/learn-blockchains-by-building-one-117428612f46) — concise and rewarding.
    - **Ruby**: [Build your own Redis](https://rohitpaulk.com/articles/redis-0) — forces you to understand TCP, concurrency, and protocol design.

    At this level, multi-part series are especially valuable because they mirror how real software evolves over time.
  </Tab>
  <Tab title="Advanced">
    For experienced engineers who want a genuine challenge, the list includes tutorials that push deep into computer science fundamentals:

    - **C/C++**: [Write an OS from scratch](https://github.com/tuhdo/os01), [Build an Interpreter](http://www.craftinginterpreters.com/), [Write a Bootloader in C](http://3zanders.co.uk/2017/10/13/writing-a-bootloader/)
    - **Rust**: [Write an OS in pure Rust](https://os.phil-opp.com/), [Writing NES Emulator in Rust](https://bugzmanov.github.io/nes_ebook/)
    - **Haskell**: [Write You a Haskell — Build a modern functional compiler](http://dev.stephendiehl.com/fun/)

    These projects assume comfort with memory management, data structures, and low-level system concepts. They are benchmarks of mastery, not introductions.
  </Tab>
</Tabs>

## About This Project

Project Based Learning is an open-source, community-driven resource maintained on GitHub under the MIT license. It was started to solve a simple problem: most learning resources teach syntax, but few teach you how to actually *build* something. Every tutorial in the list was contributed by a developer who found it genuinely useful.

The project welcomes contributions from anyone. If you have followed a great project-based tutorial that isn't listed yet, you can open a pull request to add it. See the [Contributing](/contributing) page for formatting rules and submission guidelines.
